## 전체 표준 카탈로그 무료 공개

- SMPTE는 미디어 전문가, 기술자, 엔지니어를 위한 조직으로, **전체 Standards 카탈로그**를 글로벌 미디어 기술 커뮤니티에 무료 공개함
- 포함 범위는 다음과 같음
  - 모든 published **SMPTE Standards**
  - Recommended Practices
  - Engineering Guidelines
  - Registered Disclosure Documents(RDDs)
  - 향후 공개될 모든 릴리스
- SMPTE Standards는 100년 넘게 엔터테인먼트 기술 산업의 **상호운용성**을 뒷받침해 왔음
- 접근 장벽이 낮아지면 표준 채택과 구현이 빨라지고, 다음 세대 미디어 기술 혁신을 지원할 수 있음

## 공개 결정의 배경

- SMPTE는 110년 동안 미디어 기술 산업과 함께 변화해 왔으며, 이번 무료 공개도 그 연장선에 있음
- 산업은 **IP 기반 워크플로**, **AI 진정성(authenticity)·콘텐츠 출처(provenance)** 까지 전환적 변화에 직면, 또 하나의 변곡점에 위치해 있음
- 회원, 파트너, 글로벌 표준 커뮤니티의 의견을 반영한 결과, 미디어의 미래에는 상호운용성이 필수라는 판단에 도달함
- 표준 접근을 여는 조치는 다음 세대 미디어 기술이 더 강하고 접근 가능한 기반 위에 구축되도록 하기 위한 움직임
  > "가볍게 내린 결정이 아님" "지금이 문을 열 때" - SMPTE 회장 Rich Welsh

## 표준 개발·출판 프로세스 현대화

- 오픈 액세스 Standards Library 전환은 SMPTE의 **표준 개발·출판 현대화** 작업의 일부임
- ### 최근 도입한 이니셔티브

  - 버전 관리, 이슈 트래킹, 자동화를 위한 **GitHub 기반 워크플로우** 채택
  - **구조화된 HTML 기반 저작(authoring)** 으로 전환
  - 문서 생성, 검토, 검증, 릴리스를 간소화하는 **통합 발행 파이프라인** 구현
- 공개 접근과 현대화 작업이 결합되면서 표준 개발 과정의 투명성이 높아지고, 산업 수요에 더 빠르게 대응할 수 있음
- 기존 표준의 품질과 엄격함을 유지하는 것도 함께 강조됨

- 개발자, 통합 업체, 교육자, 제조사, 신흥 시장은 2차 출처가 아니라 정확한 명세를 기반으로 구축할 수 있음

- 전체 카탈로그는 [SMPTE Standards Library](https://pub.smpte.org/doc/)를 통해 접근 가능함

## 한국 방산의 역사적 기원

- 무기 사업의 부상은 **베트남 전쟁**까지 거슬러 올라가며, Nixon은 아시아 동맹국이 미군 의존 대신 스스로 방위 책임을 더 져야 한다고 판단

  - 20년이 채 지나지 않은 한국 전쟁의 기억이 여전히 생생했음
- 한국에게 해답은 '건설'이었음
  - 16년 가까이 통치한 독재자 박정희 대통령은 **자주국방 필요성을 강조**하며 방위 산업에 막대한 자금 투자
  - 외국 무기의 라이선스 생산, 일부는 역설계 및 외국 기술 개조 방식 활용

- 현재 한국은 세계 9위 무기 수출국이며, [Stockholm International Peace Research Institute](https://www.sipri.org/sites/default/files/2026-03/fs_2603_at_2025.pdf) 기준으로 **세계에서 가장 빠르게 성장하는 무기 수출국** 중 하나임

- 시작 전에 다음 항목이 필요함
  - 실제로 손에 들고 있는 유효한 **정부 발급 사진 신분증**
  - 라이브 셀피 촬영이 가능한 휴대폰 또는 웹캠이 있는 컴퓨터
  - 보통 5분 미만의 시간
- 허용되는 신분증은 대부분 국가의 원본 물리 정부 발급 사진 신분증임
  - 여권
  - 운전면허증 또는 주·도 신분증
  - 국가 신분증
- 문서는 정부가 발급했으며, 읽기 쉽고, 손상되지 않았고, 사용자 사진을 포함해야 함
- 다음 항목은 검증에 사용할 수 없음
  - 복사본, 스크린샷, 스캔본, 사진을 다시 찍은 사진
  - 모바일 운전면허증 같은 디지털 또는 모바일 ID
  - 학생증, 직원증, 도서관 카드, 은행 카드 같은 비정부 신분증
  - 임시 종이 신분증

## 데이터 처리와 보관 책임

- Anthropic은 검증 데이터의 **데이터 컨트롤러**로서 데이터 사용 방식과 보관 기간 규칙을 정함
- 신분증과 셀피는 Anthropic 시스템이 아니라 Persona가 수집·보관하며, Persona는 Anthropic의 지시에 따라 데이터를 처리함
- Anthropic은 필요할 때 Persona 플랫폼을 통해 검증 기록에 접근할 수 있음
  - 예: 이의제기 검토
  - 신분증 이미지와 셀피 이미지를 자체적으로 복사하거나 저장하지 않음
- Persona는 계약상 검증 제공·지원과 사기 방지 능력 개선에 한해 데이터를 사용할 수 있음
  - 업계 표준 보안 통제로 데이터를 보호해야 함
  - Anthropic이 정한 보관 제한과 관련 법에 따라 데이터를 삭제해야 함
- Persona를 오가는 모든 데이터는 전송 중과 저장 시 **암호화**됨
- 개인정보 처리 세부 사항은 [Privacy Policy](https://www.anthropic.com/legal/privacy)에서 확인할 수 있음

## 검증 데이터가 사용되지 않는 곳

- 신원 데이터는 **모델 학습**에 사용되지 않으며, 본인 확인과 법적·안전 의무 충족에만 쓰임
- Anthropic은 신원 확인에 필요한 최소 정보만 요청함
- 검증 데이터는 사용자, Persona, Anthropic 사이에 머무름
  - 유효한 법적 절차에 대응해야 하는 경우는 예외임
  - 마케팅, 광고, 검증 및 컴플라이언스와 무관한 목적을 위해 제3자에게 공유되지 않음

## 실패 대응과 계정 이의제기

- 검증 실패는 흐릿한 사진, 읽을 수 없는 문서, 만료된 신분증, 기술적 문제 등에서 발생할 수 있음
- 실패 시 검증 흐름 안에서 여러 번 재시도할 수 있으며, 대부분은 더 좋은 조명에서 사진을 다시 찍거나 다른 정부 발급 사진 신분증을 사용해 해결됨
- 신분증이 손상되지 않았고 명확하게 읽히는지도 확인해야 함
- 시도 횟수를 모두 사용해도 검증할 수 없으면 [지원 양식](http://claude.com/form/identity-verification-help)을 제출할 수 있음
- 검증 후 계정이 금지될 수 있는 이유는 다음과 같음
  - 사용 정책 반복 위반
  - 지원되지 않는 지역에서의 계정 생성
  - 서비스 약관 위반
  - 18세 미만 사용
- 계정 정지 또는 종료가 잘못됐다고 판단하면 [claude.ai](http://claude.ai)에 금지된 계정으로 로그인한 뒤 [Submit an appeal](https://claude.ai/restricted) 양식을 제출해야 함
  - 이의제기 양식에 접근하려면 로그인 상태여야 함
  - Safeguards 팀이 계정 비활성화 이유를 추가 조사할 수 있음
- 신원 확인, 데이터, 검증 절차 관련 질문은 [지원 양식](http://claude.com/form/identity-verification-help)으로 문의할 수 있음

## 연구 개요와 핵심 발견

- Cornell University의 Hal Triedman, Tingwei Zhang, Vitaly Shmatikov가 작성한 프리프린트 논문 **"[Deep-research agents can be poisoned via user-generated content](https://arxiv.org/pdf/2605.24245)"**
  - 이 논문은 Reddit 모더레이터와 Wikipedia 편집자들이 이미 체감하던 문제, 즉 **AEO(AI-engine optimization)** 를 노린 브랜드 홍보성 콘텐츠 범람에 대한 메커니즘과 연구적 근거 제공
- 딥리서치 에이전트는 사용자 질의에 대해 **인용과 함께 웹 콘텐츠를 가져오는 실시간 스크래퍼**임
  - 전체 질의의 약 절반에서 Reddit·Wikipedia 같은 UGC 인용
  - 전체 인용의 약 1/4이 UGC 사이트에서 발생
- 논문은 이를 **"[피자에 풀을 바르라고 권하는 Redditor](https://www.404media.co/google-is-paying-reddit-60-million-for-fucksmith-to-tell-its-users-to-eat-glue/)"** 사례처럼 정보 접근 시스템 전반에 대한 종단(end-to-end) 공격으로 규정
  - 단일 오염 댓글이 관련 AI 질의 클러스터 전체의 출력에 영향을 줄 수 있음을 확인

## 조작이 쉬운 이유 — 어휘적 유사성

- 다수의 딥리서치 에이전트와 LLM이 정확성 대신 **질의와의 어휘적 유사성**을 정확도의 대용물로 사용함
  - LLM은 사용자가 물은 질의와 비슷하게 읽히는 콘텐츠를 반환하는 경향이 있어, AEO 수행 브랜드가 사람들이 AI에 묻는 질문을 연구해 Reddit에 유사 문구 생성 가능
- **11~15단어** 분량의 짧은 텍스트가 질의와 매우 유사할 경우 LLM에게 특히 설득력 있게 작용함
  - 보조제(supplement)를 팔려는 사람이 오염시키고 싶은 질의를 식별하고, 그와 흡사한 문구를 Reddit에 올리면 LLM에 효과적으로 침투
- 공격 방식 자체가 예상보다 훨씬 단순하며, 관련성 높은 서브레딧에 **타깃 게시 → 인기 질의에 맞춘 문구 작성 → 모더레이션 회피**의 반복으로 충분함
