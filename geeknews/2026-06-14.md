## 00:07 — 모든 프레임을 완벽하게
**source**: geeknews · **tags**: #UI #상태 #애니메이션 #전환 #사례
**url**: https://news.hada.io/topic?id=30467

- UI는 사용자가 앱 품질을 판단하는 거의 유일한 표면이며, 어느 순간 스크린샷을 찍어도 **화면 상태**가 말이 되어야 신뢰가 쌓임
- 완성도 높은 UI는 개발자가 다듬는 시간을 들였다는 신호가 되고, 코드 품질도 비슷하게 다듬었을 것이라는 **합리적 휴리스틱**이 됨
- 실제 기준은 화면 전환 사이 흰색 깜빡임, 부분 로딩, 로딩 중 재배치, 상태 문구 불일치, 부정확한 **애니메이션**을 없애는 것임
- 시작과 끝 상태가 좋아도 중간 프레임이 어긋나면 구성요소가 동기화되지 않은 느낌을 만들고, Photos 사례처럼 실제 변화가 없는 전환에서도 **거짓 감각**을 만들 수 있음
- 애니메이션은 전환 이해를 도와야 하며, 시작·끝뿐 아니라 그 사이 모든 프레임까지 관리해야 **정밀한 도구** 같은 UI가 됨

## 핵심 원칙

- Wayland의 명시 목표인 [“every frame is perfect”](https://wayland-book.com/protocol-design/design-patterns.html)는 현대 GPU 스택의 복잡성 속에서 제어권을 되찾으려는 기술적 목표임
- 같은 원칙은 UI에도 적용되며, 앱의 어느 순간을 스크린샷으로 찍어도 화면이 자연스럽고 일관되어야 함
- 사용자는 코드를 볼 수 없기 때문에 UI가 앱 품질을 판단하는 유일한 표면이 됨
- UI가 잘 다듬어져 있으면 개발자가 폴리싱 시간을 들였다는 신호가 되고, 코드도 비슷한 수준으로 다듬었을 것이라는 판단으로 이어짐

## 실제 기준과 사례

- 완벽한 프레임을 위해서는 화면 사이 **흰색 깜빡임**이 없어야 하고, 콘텐츠가 부분적으로 로딩된 상태나 로딩 중 레이아웃 재배치가 없어야 함
- UI 내부 상태는 일관되어야 하며, 한 부분이 “1 update available”이라고 말할 때 다른 부분이 “Checking for updates...”라고 말하면 안 됨
- 애니메이션은 자주 잊히며, 시작 상태와 끝 상태가 좋아도 그 사이가 어색하면 중간 스크린샷이 말이 안 되는 프레임이 됨
- Safari 사례에서는 플레이스홀더 텍스트가 가운데에서 움직이지만 커서는 왼쪽 위치에서 애니메이션되어 두 구성요소가 서로 동기화되지 않은 느낌을 만듦
- Photos 사례에서는 Crop과 Adjust 모드를 전환할 때 사진은 즉시 제자리로 이동하지만 크롭 테두리는 애니메이션되어, 모드 전환 중 무언가 미묘하게 바뀌는 **거짓 느낌**을 만듦
- 검색 UI 사례에서는 전환 이해를 도와야 할 애니메이션이 오히려 돋보기 움직임을 따라가기 어렵게 만듦
- Youtube 사례에서는 직사각형을 한 위치에서 다른 위치로 옮기는 단순한 작업에서도 이상한 움직임이 생기며, 이유와 관계없이 결과는 완벽하지 않은 프레임이 됨
- Preview 앱의 확대 애니메이션까지 포함해 핵심은 시작과 끝 상태뿐 아니라 그 사이 모든 순간에도 주의를 기울여야 한다는 점임

## 00:17 — 거절된 이모지 제안들
**source**: geeknews · **tags**: #제안 #문서 #항목 #범주 #결정
**url**: https://news.hada.io/topic?id=30468

- 공식 문자 제안서에 오른 뒤 새 문자 추가로 이어지지 않은 **이모지 후보**를 범주별 표로 모은 목록임
- 대상은 Unicode Consortium에 제출된 **공식 문자 제안서**의 이모지이며, 개요 문서의 논의나 비공식 요청은 집계하지 않음
- 공식 불승인 공지가 없어도 새 문자 추가로 이어지지 않은 제안은 거절로 간주되며, 목록은 **100% 완전하지 않을 가능성**이 높음
- 얼굴, 손동작, 음식, 동식물, 사람, 사물, 기호, 깃발, modifier 등 여러 범주에서 실제 제안 문서와 **decision** 항목이 연결됨
- 일부 후보는 승인 과정에서 다른 문자나 시퀀스로 대체됐고, 일부 문자는 Universal Character Set에 들어갔지만 **이모지 지위**를 받지 못함

---

## 목록의 범위와 판정 기준

- **공식 문자 제안서**에 포함된 이모지만 목록 대상으로 삼으며, 가능한 추가 항목을 논의한 개요 문서나 비공식 요청은 집계하지 않음
- 이모지로 쓰일 의도로 제안된 기호만 고려하지만, 일부 항목은 해당 지위가 불명확할 수 있음
- 한때 제안됐지만 다른 제안서나 약간 다른 형태로 최종 추가된 이모지는 목록에서 제외됨
- 공식 불승인 공지가 없더라도 새 문자 추가로 이어지지 않은 제안은 거절된 것으로 간주됨
- 목록은 완전하지 않을 가능성이 높으며, 실패한 이모지 제안의 더 완전한 열거는 Unicode Consortium의 Emoji Proposals Status 스프레드시트를 참조해야 함

## 표의 구성과 표시 방식

- 표는 **Sample, Name, Author, Date, Document, Decision** 열로 거절된 이모지 제안을 정리함
- 샘플 이미지는 대개 제안서에서 직접 가져오며, 항상 모범 사례를 정확히 반영하지는 않을 수 있음
- 제안 문서에 적절한 그림이 없으면 **sample** 열이 비어 있음
- **decision** 열은 해당 이모지의 거절과 관련된 문서와 action item을 나열함
- 거절 관련 논의는 매우 모호할 수 있으며, 일부 논의는 같은 제안서 안의 다른 문자만 다뤄 목록의 이모지에 대한 결정을 암시할 수 있음
- 제안의 존재만 인정하고 입장을 밝히지 않은 노트는 별표(\*)로 표시되며, 제안자에게 비공개로만 전달된 결정은 dagger(†)로 표시됨
- **withdrawn** 상태는 해당 이모지가 문서의 이전 반복본에만 있었고 이후 제안자들이 직접 대체 문서를 냈음을 뜻함

  - service animal vest 제안은 승인 과정에서 🦺로 대체됨
  - Other Symbols 범주는 Emoji Compatibility Symbol-1부터 Emoji Compatibility Symbol-66까지의 묶음, U+2B95 ⮕ Rightwards Black Arrow, U+221E ∞ Infinity, Pi, Heart with Knife ZWJ sequence “❤️ + 🔪 = ❤️‍🔪”를 포함함
  - Heart with Knife는 2017-06-28 [L2/17-228](https://www.unicode.org/L2/L2017/17228-heart-with-knife-emoji.pdf)로 제안됐고, 결정 항목으로 [L2/17-380](https://www.unicode.org/L2/L2017/17380-emoji-sc-report-q3.pdf)와 [L2/17-401](https://www.unicode.org/L2/L2017/17401-ad-hoc-recs-emoji-6.pdf)이 연결됨
  - ♾️는 공식적으로 infinity emoji로 사용되지만, 실제로는 permanent paper sign임

## Flags, Modifiers, Unsorted

- ### Flags

  - 미국 주 깃발 제안은 2016-08-13에 제출된 항목이며, Alabama부터 Wyoming까지 50개 주별 Flag 이모지가 **tag sequence** 형식으로 정리됨
  - 미국 주 깃발 항목은 Alabama `usal`, Alaska `usak`, Arizona `usaz`, Arkansas `usar`, California `usca`, Colorado `usco`, Connecticut `usct`, Delaware `usde`, Florida `usfl`, Georgia `usga`, Hawaiʻi `ushi`, Idaho `usid`, Illinois `usil`, Indiana `usin`, Iowa `usia`, Kansas `usks`, Kentucky `usky`, Louisiana `usla`, Maine `usme`, Maryland `usmd`, Massachusetts `usma`, Michigan `usmi`, Minnesota `usmn`, Mississippi `usms`, Missouri `usmo`, Montana `usmt`, Nebraska `usne`, Nevada `usnv`, New Hampshire `usnh`, New Jersey `usnj`, New Mexico `usnm`, New York `usny`, North Carolina `usnc`, North Dakota `usnd`, Ohio `usoh`, Oklahoma `usok`, Oregon `usor`, Pennsylvania `uspa`, Rhode Island `usri`, South Carolina `ussc`, South Dakota `ussd`, Tennessee `ustn`, Texas `ustx`, Utah `usut`, Vermont `usvt`, Virginia `usva`, Washington `uswa`, West Virginia `uswv`, Wisconsin `uswi`, Wyoming `uswy` tag sequence를 사용함
  - 미국 주 깃발 제안의 문서 링크는 Unicode 문서 [L2/17-382](https://www.unicode.org/L2/L2017/17382-rgi-flag-omnibus.pdf#page=37)로 표시됨
  - Flag for Northern Ireland는 tag sequence `gbnir`을 사용하고, 문서 링크는 [L2/17-382](https://www.unicode.org/L2/L2017/17382-rgi-flag-omnibus.pdf#page=29)로 표시됨
  - Flag for Basque Country는 tag sequence `espv`, Flag for Brittany는 `frbre`, Flag for Catalonia는 `esct`, Flag for Frisia는 `nlfr`를 사용함
  - Flag for Frisia는 문서 링크 [L2/18-233](https://www.unicode.org/L2/L2018/18233-frisian-flag-emoji.pdf)와 결정 링크 [L2/18-222R3](https://www.unicode.org/L2/L2018/18222r3-esc-uni12-recmd-q3.pdf)가 표시됨
- ### Modifiers

  - Modifiers 섹션에는 Image Hash-0000부터 Image Hash-FFFF까지 임의 이미지의 coded hash를 다루는 제안이 있음
  - Image Hash 제안은 2016-05-02에 제출됐고, 문서 링크는 [L2/16-105](https://www.unicode.org/L2/L2016/16105-unicode-image-hash.pdf)임
  - Grey 색상 modifier 제안은 2016-10-31에 제출됐고, 문서 링크는 [L2/16-318](https://www.unicode.org/L2/L2016/16318-ten-colors.pdf)임
  - Emoji Modifier Colour Grey, Emoji Modifier Colour Dark Blue, Emoji Modifier Colour Light Blue, Emoji Modifier Colour Pink 제안은 2017-08-19에 제출됐고, 문서 링크는 [L2/17-355](https://www.unicode.org/L2/L2017/17355-kendal-colour-emoji.pdf)임
- ### Unsorted

  - Unsorted 섹션에는 Period, Bloated, Fibonacci Series 항목이 있음
  - Period 항목 중 하나는 “panty with blood stain” 의미이며, 2016-03-21 [L2/16-126](https://www.unicode.org/L2/L2016/16126-menstrual-cycle.pdf)에 연결됨
  - 또 다른 Period 항목은 2017-06-30 [L2/17-216](https://www.unicode.org/L2/L2017/17216-period-emoji.pdf)에 연결됨
  - Bloated 항목은 Period의 “panty with blood stain” 제안과 같은 문서 [L2/16-126](https://www.unicode.org/L2/L2016/16126-menstrual-cycle.pdf)에 연결됨
  - Fibonacci Series 항목은 2017-04-18 [L2/17-113](https://www.unicode.org/L2/L2017/17113-science-emoji.pdf)에 연결되며, 결정 링크로 [151-A100](https://www.unicode.org/L2/L2017/17103.htm#151-A100)과 [L2/17-131](https://www.unicode.org/L2/L2017/17131-esc-report.pdf)이 표시됨

## Universal Character Set에는 들어갔지만 이모지가 되지 않은 문자

- Universal Character Set에 추가됐지만 **emoji status**를 받지 못한 문자들이 있음
- 이 문자들은 이모지로 사용될 의도로 제안됐으나 최종적으로 symbol로만 수용됨
- U+1F6D3 🛓 Stupa와 U+1F6D4 🛔 Pagoda는 2015-01-11에 제안됐고, 문서는 L2/15-007임
- U+1F93B 🤻 Modern Pentathlon과 U+1F946 🥆 Rifle은 2015-07-31에 제안됐고, 문서는 L2/15-196R4임
- 공식 제출된 모든 이모지 제안이 Unicode Technical Committee까지 도달하는 것은 아님
- 시간이 지나며 Emoji Subcommittee가 document registry에 실제로 게시하는 제안 비율이 꾸준히 줄어들었고, 그 결과 실패한 이모지 제안의 대부분을 이 페이지에서 수집하는 것은 불가능함
